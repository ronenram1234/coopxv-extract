# Feature Development Workflow

## Phase 1: Requirements (Shira leads)
1. Shira analyzes from `docs/input/requirements/`
2. Creates spec in `docs/output/requirements/`
3. Ronen gets approval

## Phase 2: Architecture (Gil leads)
1. Gil reviews Shira spec
2. Designs solution in `docs/output/architecture/`
3. Ronen gets approval

## Phase 3: Infrastructure (Michael leads)
1. Michael sets up infrastructure
2. Configures environments

## Phase 4: Implementation (Parallel)
- Maya: UI/UX
- Backend: Services/APIs

## Phase 5: Testing (Lisa leads)
1. Creates tests
2. Verifies coverage
3. Language QA

## Phase 6: Review (Alex leads)
1. Code review
2. Security audit
3. Approval

## Phase 7: Deployment (Michael leads)
1. Deploy staging
2. Get approval
3. Deploy production

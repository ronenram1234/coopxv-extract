1. סקירה כללית — Overview

שירות ה-Extract מנטר קבצי Excel (בתבנית *.xlsx) מתיקיית המקור, קורא את הגיליון BData_in, ומזהה שורות פעילות (עמודה A=1). השירות מזהה שורות חדשות ושורות שהשתנו לעומת הסריקה הקודמת, ומכניס אותן ל-MongoDB.

עיקרון מפתח: שורה נחשבת "מעודכנת" כאשר מספר השורה זהה לסריקה הקודמת, אך תוכן העמודות השתנה — השוואה טקסטואלית, עמודה-עמודה.

2. תהליך הסריקה — Scan Flow

סריקת תיקייה לקבצי *.xlsx
▼
קריאת גיליון BData_in בלבד
▼
סינון שורות: עמודה A = 1 (פעילות)
▼
טעינת שורה אחרונה שנשלפה מ-DB
▼
בחירת שורה הבאה: חדשה או מעודכנת?
▼
השוואת תוכן — שונה? → הכנסה | זהה? → דילוג
▼
עדכון סטטוס להקה ב-MongoDB

3. כללי עמודה A — Column A Classification

ערך בעמודה A משמעות פעולה סטטוס
1 שורה פעילה — מועמדת לחילוץ INSERT אם חדשה/שונתה active
11 דגל תחזוקה — נספרת בנפרד COUNT ONLY maintenance
אחר שורה סגורה / לא רלוונטית IGNORE closed

סטטוס להקה (Flock Status)

תנאי סטטוס
יש שורות עם A=1active
אין A=1, יש A=11maintenance
אין A=1 ואין A=11closed

4. זיהוי שורה מעודכנת — Updated Line Detection

עץ החלטות — Decision Tree

┌─ Row number > lastProcessedRowNumber? │ └─ YES → INSERT AS NEW │ ├─ Row number == lastProcessedRowNumber? │ └─ Compare content with areRowsEqualAsText() │ ├─ Content IDENTICAL → SKIP (unchanged) │ └─ Content CHANGED → INSERT AS UPDATED │ └─ No candidates (A=1) found? └─ SKIP (no rows to process)

סדר עדיפויות — Priority

  1. שורה חדשה (rowNumber גבוה יותר) — עדיפות ראשונה
  2. אותה שורה — תוכן שונה — עדיפות שנייה (שורה מעודכנת)
  3. אותה שורה — תוכן זהה — דילוג

5. לוגיקת השוואה — Content Comparison

הפונקציה areRowsEqualAsText() משווה בין השורה השמורה ב-DB לבין השורה הנוכחית מהקובץ.

שורה שמורה (DB)

normalizeStoredDataToText(doc.data) → { A: "1", B: "אורטל", C: "3", ... }

שורה נוכחית (Excel)

buildDataAndMetadataFromRow(row) → { A: "1", B: "אורטל", C: "4", ... }

כללי השוואה

כלל פירוט
נורמליזציה כל ערך מומר ל-String ועובר .trim()
השוואה עמודה-עמודה כל העמודות משני המקורות (union) נבדקות
רגיש לאותיות Case-sensitive — "ABC" ≠ "abc"
עמודות ריקות בסוף מתעלמים מעמודות שריקות בשני הצדדים
הבדל אחד = שינוי מספיק שעמודה אחת שונה → השורה "מעודכנת"
שימו לב: גם הבדל קטן כמו רווח נוסף או שינוי באות גדולה/קטנה ייחשב כשינוי ויגרום להכנסת שורה חדשה.

7. שמירה ב-MongoDB — Data Persistence

אוספים (Collections)

Collection תיאור מפתח
extracted_lines שורות שנשלפו — כל שורה חדשה/מעודכנת scanId + filename + rowNumber
scans מטא-דאטה של כל סריקה — סטטיסטיקות, קבצים שעובדו scanNumber + timestamp
flock_status סטטוס להקה לפי קובץ — active / maintenance / closed filename (upsert)
